Types of Database
  • Relational Database (RDBMS) - here Data stored in tables. ex - MySQL, Oracle, DB2, SQL Server, MS Access, postgresSQL, etc
    Data is stored in tables (rows & columns).
    Tables are connected using relations (keys).
    Uses SQL (Structured Query Language).

  • NoSQL Database (Non-Relational Database) - here data not stored in tables. ex - MongoDB, CouchDB, Cassandra, etc pair
    Non-relational (no fixed tables or relations).
    Flexible structure ‚Äî JSON, key-value, document, etc.
    Used for big data, real-time apps, or unstructured data.

Foreign Key - A foreign key is a column (or set of columns) in a table that refers to the primary key in another table. FKs can have duplicate & null values.
There can be multiple FKs.
to prevent actions that would destroy links between tables.(MySQL stops you from deleting or updating data that other tables still depend on.)

Cascading for Foreign key

On Delete Cascade : When we create a foreign key using (Delete Cascade) this option, it deletes the referencing rows in the child table when the referenced row is deleted in the parent table which has a primary key.


On Update Cascade : When we create a foreign key using UPDATE CASCADE the referencing rows are updated in the child table when the referenced row is updated in the parent table which has a primary key.

SQL Datatypes - signed and unsigned

this is another important SQL concept, especially when dealing with numeric data types like INT, SMALLINT, BIGINT, TINYINT etc.
In SQL (especially MySQL), signed and unsigned define whether a numeric column can store negative

TINYINT UNSIGNED (0 to 255)
TINYINT SIGNED (-128 to 127)
INT SIGNED (-2147483648 to 2147483647)
INT UNSIGNED (0 to 4294967295)

You need negative numbers (like temperature, profit/loss) - SIGNED
You only store positive values (like age, price, quantity, ID, quantity, weight) - UNSIGNED

| Type | Meaning | Example |

| ------------ | -------------------------------------------------------- | --------------- |

| **SIGNED** | Can store both **positive and negative** numbers | -100, 0, +100 |

| **UNSIGNED** | Can store **only positive** numbers (no negative values) | 0, 10, 500, 999 |


difference between char and varchar:

| Feature | `CHAR(n)` | `VARCHAR(n)` |

| ---------------------- | --------------------------------------------------------------- | -------------------------------------------------------- |

| **Full form** | Character | Variable Character |

| **Storage Type** | Fixed-length | Variable-length |

| **Storage Behavior** | Always reserves `n` characters (pads with spaces if needed) | Stores only the actual number of characters used |

| **Performance** | Slightly faster (fixed size) | Slightly slower (variable size) |

| **Use Case** | When data length is constant (e.g., country codes, PIN, gender) | When data length varies (e.g., names, emails, addresses) |

| **Example (n = 10)** | `'abc'` ‚Üí stored as `'abc_______'` (7 spaces) | `'abc'` ‚Üí stored as `'abc'` (3 chars only) |

| **Storage Space Used** | Always uses `n` bytes | Uses length of data + 1 or 2 bytes overhead |

| **Trailing Spaces** | Padded automatically (and usually ignored in comparisons) | Not padded automatically |

Clauses A clause is a keyword used inside an SQL query to perform a specific task.
List of Main SQL Clauses
SELECT Clause
Used to choose the columns you want.
      SELECT name, age FROM employee;
Get all columns:
      SELECT * FROM employee;
FROM Clause
Tells SQL which table to read data from.
      SELECT * FROM customers;

WHERE Clause
Used to apply conditions (filter data).
      SELECT * FROM customers WHERE address = 'Park Lane 38';
      SELECT * FROM employee WHERE age > 25;

      Operators you can use:
      =, !=
      >, <, >=, <=
      BETWEEN
      LIKE
      IN
      IS NULL
      IS NOT NULL
      AND
      OR
      NOT
      SELECT * FROM employee WHERE name LIKE 'A%';

ORDER BY Clause
      Used to sort data. by default it is ascending
      SELECT * FROM employee ORDER BY name;
      SELECT * FROM employee ORDER BY age ASC;;
      SELECT * FROM employee ORDER BY name DESC;
      select * from employee order by age desc, name asc limit 3;
multi-level Sorting: Sort students by marks highest ‚Üí lowest. If two students have the same marks, then sort them by name A ‚Üí Z. This is called multi-level sorting.


GROUP BY Clause
      Used to group rows by one or more columns. Used to group rows that have the same values.
      SELECT * FROM employee GROUP BY age;
      SELECT * FROM employee GROUP BY age, name;
      SELECT department, COUNT(*) FROM employee GROUP BY department;

HAVING Clause
      Used to filter groups created by the GROUP BY clause.
      Used to apply conditions after grouping. (WHERE cannot be used after GROUP BY)
      SELECT department, COUNT(*) FROM employee GROUP BY department;
      SELECT department, COUNT(*) FROM employee GROUP BY department HAVING COUNT(*) > 1;

     Get departments with more than 5 employees:
     SELECT department, COUNT(*)
     FROM employee
     GROUP BY department
     HAVING COUNT(*) > 5;


Limit Clause
      Used to limit the number of rows returned by an SQL statement.
      select * from student where marks > 75 limit 5;
      SELECT * FROM employee LIMIT 5;
      SELECT * FROM employee LIMIT 5 OFFSET 2;
      In SQL Server, use: TOP
      SELECT TOP 5 * FROM employee;

JOIN Clause

      Used to combine rows from two or more tables.

What Are SQL Functions?
     SQL functions are built-in operations that perform calculations, transform data, or summarize results.
     SQL functions are used to perform operations on data.

           Scalar functions (concat, substring, upper, lower, length, round, ceil, floor) : - Work on one value and return one value.

Average marks per department:       SELECT dept, AVG(marks) FROM student GROUP BY dept;
Minimum marks per department:       SELECT dept, MIN(marks) FROM student GROUP BY dept;
Maximum marks per department:       SELECT dept, MAX(marks) FROM student GROUP BY dept;

‚≠ê Using HAVING with Aggregate Functions
Get departments with more than 5 employees:
SELECT department, COUNT(*)
FROM employee
GROUP BY department
HAVING COUNT(*) > 5;

Get departments with average marks > 85:
SELECT department, AVG(marks)
FROM student
GROUP BY department
HAVING AVG(marks) > 85;


SQL Sub Queries A Subquery or Inner query or Nested query
A Subquery or Inner query or a Nested query is a query within another SQL query.
It is used when you need a result from another query to complete the main query.
üëâ A subquery always returns some value(s), and the outer query uses those values.

Types of Subqueries

  • Single-row Subquery
  • A subquery that returns only one value.
    Example:
    Find students who have the highest marks.

    SELECT name, marks
    FROM student
    WHERE marks = (SELECT MAX(marks) FROM student);

    Here:
    Inner query ‚Üí gets highest marks
    Outer query ‚Üí finds the student with that mark

  • Multi-row Subquery
    A subquery that returns multiple rows, used with:

    • IN
    • ANY
    • ALL
    • EXISTS
    • NOT EXISTS


    Find teachers who belong to departments located in 'Delhi'.

    SELECT name
    FROM teacher
    WHERE department_id IN
    (SELECT id
    FROM department
    WHERE location = 'Delhi');

    Inner query returns multiple dept IDs ‚Üí outer query uses them.

  • Multi-column Subquery
    Subquery returns more than one column.
    Example:
    Find departments where (min_salary, max_salary) match the company standard.

    SELECT *
    or
    SELECT dept_name
    FROM department
    WHERE (min_salary, max_salary) IN
    (SELECT min_salary, max_salary
    FROM company_standard);


  • Nested Subquery (inside WHERE, FROM, SELECT)
    • WHERE wtih EXISTS clause:
      EXISTS checks if the subquery returns at least one row (TRUE/FALSE).
      Example:
      Find departments that have at least one teacher.

      SELECT dept_name
      FROM dept d
      WHERE EXISTS ( SELECT 1 FROM teacher t WHERE t.dept_id = d.id);


    • FROM
      A subquery used as a table.
      Example:
      Find average marks of top 5 students.

      SELECT AVG(marks) FROM (SELECT marks FROM student ORDER BY marks DESC LIMIT 5) AS top5;
      The inner query creates a temporary table top5.


    • SELECT
      Useful for adding calculated fields

      Example:
      Show each student name with total student count.
      SELECT name,
      (SELECT COUNT(*) FROM student) AS total_students
      FROM student;

      Same count repeated for each row (useful in reports).
